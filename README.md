# @sameer405/ui

RapidCanvas UX Design Library — 23 React components + design tokens, built from
Figma (`F7YIPUOWnqIZAOcnFszBcr`). This package is the **single source of truth**
for the design system; the Storybook here is its living documentation.

Components are styled with **Tailwind CSS**. They ship as compiled ES modules,
but the Tailwind utility classes are generated by the *consuming* app — so a
consumer must run Tailwind and include this package in its `content` globs.

## Components

`AnimatedCalendar`, `Badge`, `Button`, `Checkbox`, `ErrorBoundary`, `IconButton`,
`InputField`, `LinkText`, `Menu` (`MenuItem`/`MenuGroup`), `Notification`
(`NotificationItem`/`NotificationsPanel`), `Pagination`, `ProgressBar`,
`RadioButton`, `SideNav`, `Slider`, `StatusDotBlinker`, `Switch`, `Tabs`, `Tag`,
`ToastNotification`, `Toggle`, `Tooltip`, `TopNav`.

Plus tokens: `colors`, `spacing`, `radius`, `typography`, `shadows`.

## Develop

```bash
npm install
npm run storybook     # http://localhost:6006
npm run build         # emit dist/ (ESM + .d.ts)
npm run typecheck
```

## Publish (GitHub Packages)

This package publishes to **GitHub Packages** under the `@sameer405` scope, which
matches the repo owner (`github.com/sameer405/ui_storybook`) — a requirement of
GitHub Packages. `repository` in `package.json` points at that repo.

Config is already in place:
- `.npmrc` routes `@sameer405` → `https://npm.pkg.github.com` and reads the token
  from `$NODE_AUTH_TOKEN`.
- `publishConfig` in `package.json` pins the publish registry.

### Automated (recommended)

`.github/workflows/publish.yml` builds and publishes whenever you create a
**GitHub Release** (or run the workflow manually). It uses the built-in
`GITHUB_TOKEN` — no extra secrets needed. If this package lives in a subdirectory
of the repo, uncomment the `working-directory` block in that workflow.

```bash
npm version patch     # bump + commit + tag
git push --follow-tags
# then publish a Release for that tag in the GitHub UI → workflow runs
```

### Manual (from your machine)

```bash
export NODE_AUTH_TOKEN=<GitHub PAT with write:packages scope>
npm version patch
npm publish           # prepublishOnly runs the build first
```

## Consume in another project

1. Tell the consuming project where the `@sameer405` scope lives. Add an `.npmrc`
   at its repo root (the token can come from an env var so it isn't committed):

   ```ini
   @sameer405:registry=https://npm.pkg.github.com
   //npm.pkg.github.com/:_authToken=${NODE_AUTH_TOKEN}
   ```

   For a **private** package the consumer must export a `NODE_AUTH_TOKEN` (a
   GitHub PAT with at least `read:packages`) before installing. In CI, use
   `actions/setup-node` with `registry-url: https://npm.pkg.github.com` and
   `scope: '@sameer405'`, passing `NODE_AUTH_TOKEN: ${{ secrets.GITHUB_TOKEN }}`.

2. Install:

   ```bash
   npm install @sameer405/ui
   ```

3. Add the preset and the package's dist to your Tailwind config so the utility
   classes used inside the components actually get generated:

   ```js
   // tailwind.config.js
   module.exports = {
     presets: [require('@sameer405/ui/tailwind-preset')],
     content: [
       './src/**/*.{ts,tsx}',
       './node_modules/@sameer405/ui/dist/**/*.js', // <- required
     ],
   };
   ```

4. Use components:

   ```tsx
   import { Button, Badge, Tabs } from '@sameer405/ui';

   export function Example() {
     return <Button variant="primary" size="md">Save</Button>;
   }
   ```

> **Note on theming:** some components use hardcoded brand hex (e.g.
> `bg-[#ED600E]`). They render correctly without the preset, but to make the
> palette themeable downstream, migrate those to token classes (`bg-primary`)
> over time.
